## 1. 이 문서는 무엇인가요?

**Tech Market Intelligence Monitor**(이하 **모니터**)는 프라운호퍼 한국 사무소용 **기술 시장·R&D 타겟팅 모니터링 자동화 도구**입니다.

프로그램이 국내 RSS에서 기사·보도자료를 수집하고, 정부·공공기관 URL은 본문·PDF까지 보강한 뒤, 키워드·**정부·R&D 타깃** 규칙으로 걸러 AI 요약합니다. 결과는 GitHub 저장소에 올라갑니다.

| 단계 | 하는 일 |
|------|---------|
| 1. 수집 | `sources.txt` 국내 RSS 피드에서 기사·보도자료 목록 가져오기 |
| 2. 보강 | 정부·공공기관 페이지 본문·첨부 PDF 추출 (요약 품질 향상) |
| 3. 필터 | `keywords.txt` 매칭 + 정부·R&D 타깃 기사는 키워드 없이도 통과 |
| 4. 요약 | AI가 **투자 주체·목적·위탁 연구 니즈·접근 전략** 중심 한국어 요약 작성 |
| 5. 저장 | **데일리 R&D 인텔리전스 로그**(`daily_YYYY-MM-DD.md`)를 저장소에 커밋 |
| 6. 월말 | 한 달치를 모아 **국내 R&D 인텔리전스 월간 보고서**(`monthly_YYYY-MM.md`) 생성·커밋 |

**GitHub 버전 사용설명서**는 위 결과물을 **브라우저에서 찾고 읽는 방법**에 초점을 둡니다.  
PC에 Python을 설치하거나 Cursor를 켤 필요는 없습니다.

## 2. GitHub만으로 충분한가요?

**리포트를 읽고 다운로드하는 용도라면 GitHub만으로 충분합니다.**

| 필요한 것 | 용도 |
|-----------|------|
| **웹 브라우저** (Chrome, Edge 등) | 저장소 접속, md 미리보기·다운로드 |

**GitHub만으로는 할 수 없는 것:**

- API 키·Secrets 최초 등록 → **저장소 관리자** 1회 설정 필요 ([§8](#8-저장소-관리자-최초-설정))
- 키워드·RSS 변경 후 **즉시 반영 실행** → Actions 수동 실행 또는 관리자 협조
- 로컬 OneDrive 폴더와 **양방향 동기화** → 로컬 PC 설정 시 [`사용설명서.md`](사용설명서.md) 참고

## 3. 무엇이 어디에 저장되나요?

저장소 루트에서 아래 폴더를 열면 됩니다.

| 결과물 | GitHub 경로 | 형식 | 용도 |
|--------|-------------|------|------|
| **데일리 R&D 인텔리전스 로그** | [`output/daily/`](output/daily/) | Markdown (`.md`) | 매일 팀이 읽는 **일일 브리핑** (R&D 기회 스캔 표) |
| **월간 R&D 인텔리전스 보고서** | [`reports/monthly_YYYY-MM.md`](reports/) | Markdown | 월간 국내 R&D·협력 기회 (적합 4점+) |

### 파일 이름 규칙

- `output/daily/daily_2026-06-29.md` → **2026년 6월 29일** 하루치 수집·요약
- `reports/monthly_2026-06.md` → **2026년 6월** 월간 R&D 인텔리전스 보고서

### 바로가기 URL 예시

```
https://github.com/yenaalisonhong/techmarketmonitor/tree/main/output/daily
https://github.com/yenaalisonhong/techmarketmonitor/tree/main/reports
```

> **팁:** GitHub에서 폴더를 연 뒤 파일명으로 정렬하면 최신 날짜가 아래쪽에 보입니다.

## 4. 데일리 리포트 읽기 (웹 브라우저)

### 4-1. 파일 열기

1. 브라우저에서 저장소 접속  
2. **`output` → `daily`** 폴더 이동  
3. 읽고 싶은 **`daily_YYYY-MM-DD.md`** 클릭  
4. GitHub가 Markdown을 **자동으로 렌더링**해 표·제목·링크가 보기 좋게 표시됩니다.

### 4-2. 문서 구조

#### (1) 상단 메타데이터

```text
날짜: 2026-06-29
기록자: Tech Market Monitor (auto)
총 항목 수: 22건 (기사 7 / 논문 15)
신뢰도 분포: A 0건 / B 22건 / C 0건
```

- **총 항목 수:** 그날 키워드에 걸려 AI 요약까지 된 기사·논문 수  
- **신뢰도:** A(최상) / B(보통·프리프린트 등) / C(참고용, 자동 생성 시 거의 없음)

#### (2) 오늘의 R&D 인텔리전스 요약

- 항목이 **1건 이상**이면 생성됩니다.  
- **모니터링 키워드**, **오늘의 공통 흐름**, **R&D 기회 스캔** 표(적합도·고객 타겟·연계 포인트)가 들어 있습니다.  
- **R&D 적합도** 1–5점 — **4점 이상**이 월간 보고서 포함 기준입니다.  
- 표의 `[제목](#anchor)` 링크를 클릭하면 **같은 페이지 아래 해당 기사 블록**으로 이동합니다.  
- **국내 R&D 타겟 시사점 (프라운호퍼)** — 국내 투자 주체 협력 관점 요약.  
- **바쁠 때는 R&D 기회 스캔 표만** 읽어도 충분한 경우가 많습니다.

#### (3) 항목 기록 (개별 기사·보도자료)

항목 제목은 **`HH:MM 제목`** 형식(발행 시각)입니다.

| 필드 | 의미 |
|------|------|
| **자료유형** | 기사 / 논문 / 보고서(시장조사) / 공식발표 등 |
| **출처** | 연합뉴스, 과기정통부, arXiv 등 |
| **링크/DOI** | 원문 URL — 클릭해서 전체 기사 확인 |
| **요약** | 한국어 팩트 bullet + `(해석)` 줄 (명사형 종결) |
| **R&D 적합도** | 1–5점 (4점+ → 월간 후보) |
| **R&D 타겟팅 (프라운호퍼)** | 제안 R&D 영역·팩트 근거·투자 주체·목적·위탁 연구 니즈·접근 전략 |
| **신뢰도** | A 또는 B (프리프린트는 별도 표기) |
| **태그** | #투자 #규제 #시장수치 등 |
| **비고** | 정부·R&D 타깃 우선, 키워드 관련도(직접/간접), 매칭 키워드 |

#### (4) 파일 하단

태그 분류체계·신뢰도 등급 기준 표가 포함됩니다.

### 4-3. GitHub에서 읽을 때 팁

| 방법 | 설명 |
|------|------|
| **렌더링 보기** | 파일 클릭만 하면 됨 (별도 확장 프로그램 불필요) |
| **원문 URL** | `링크/DOI` 항목의 URL 클릭 |
| **Executive Summary → 항목 이동** | 요약 표의 `[제목](#anchor)` 링크 클릭 |
| **인쇄·PDF** | 브라우저 인쇄(Ctrl+P) → PDF로 저장 |
| **팀 공유** | 파일 URL을 Teams·이메일에 붙여넣기 |

### 4-4. Raw(원문) 다운로드

파일 상단 **Raw** 버튼 → 전체 텍스트 다운로드 또는 다른 도구(Obsidian, Notion 등)로 가져오기.

## 5. 월간 R&D 인텔리전스 보고서 받기

### 5-1. 파일 찾기

1. 저장소에서 **`reports`** 폴더 열기  
2. **monthly_YYYY-MM.md** (예: monthly_2026-06.md) 선택

### 5-2. 읽기·다운로드

1. .md 파일 클릭 → GitHub가 **Markdown을 자동 렌더링**  
2. **Raw** 또는 **Download raw file**로 로컬 저장

### 5-3. 보고서 특징

- **R&D 적합 4점 이상** 항목만 월간 집계  
- **Executive Summary**, **Opportunities**, **Action Plan**, **R&D 스코어카드**  
- 국내 정부·기업 위탁 R&D·협력 기회 중심  

> 레거시 Word TMR은 기본 생성되지 않습니다 (GENERATE_LEGACY_TMR=1 시에만).

## 6. 자동 실행 일정

GitHub Actions가 저장소에서 **매일·매월** 파이프라인을 실행하고, 결과를 **자동으로 커밋·push**합니다.

| 워크플로 | 실행 시각 (KST) | 하는 일 |
|----------|-----------------|---------|
| **Daily Tech Market Monitor** | 매일 **08:00** | `daily-catchup` → `output/daily/*.md`, DB, 상태 파일 커밋 |
| **Monthly Tech Market Report** | 매일 **08:30** (조건부) | 해당 월 **마지막 영업일**에만 `monthly_YYYY-MM.md` 생성·커밋 |

### 마지막 영업일이란?

- 해당 월 **말일**이 토·일이면 **금요일**로 당겨 실행  
- 공휴일 캘린더는 사용하지 않음 (토·일만 제외)

### 실행 여부 확인

1. 저장소 상단 **Actions** 탭 클릭  
2. 왼쪽에서 **Daily Tech Market Monitor** 또는 **Monthly Tech Market Report** 선택  
3. 최근 실행 목록에서 ✅(성공) / ❌(실패) 확인  
4. 실행 항목 클릭 → 각 step 로그에서 상세 내용 확인

> **참고:** 스케줄(cron) 실행은 GitHub 서버 부하에 따라 **몇 분 지연**될 수 있습니다. 08:00 KST 직후 5~15분 뒤에 커밋이 올라오는 경우가 있습니다.

## 7. 수동으로 지금 실행하기

특정 날짜를 다시 만들거나, 월간 보고서를 즉시 생성하려면 **Actions**에서 수동 실행합니다.

### 7-1. 데일리 리포트 수동 실행

1. **Actions** → **Daily Tech Market Monitor**  
2. 오른쪽 **Run workflow** 클릭  
3. **Branch:** `main` 선택  
4. **Target date (YYYY-MM-DD):**  
   - 비워 두면 → `daily-catchup` (빠진 날짜 전부 보충)  
   - 날짜 입력 (예: `2026-06-20`) → 해당 날짜 **1회** 실행  
5. **Run workflow** 확인

### 7-2. 월간 보고서 수동 실행

1. **Actions** → **Monthly Tech Market Report**  
2. **Run workflow** 클릭  
3. **Year / Month:**  
   - 비워 두면 → **이번 달** 보고서  
   - 입력 (예: Year `2026`, Month `6`) → **2026년 6월** 보고서  
4. **Run workflow** 확인

> 수동 실행(`workflow_dispatch`)은 **마지막 영업일 검사 없이** 바로 월간 보고서를 생성합니다.

### 7-3. 실행 후 결과 확인

- Actions 실행이 ✅로 끝나면 **Code** 탭 → `output/daily/` 또는 `reports/`에서 새 파일 확인  
- 커밋 메시지 예: `report: daily 2026-06-29`, `report: monthly 2026-06`  
- 커밋 작성자: `github-actions[bot]`

## 8. 저장소 관리자: 최초 설정

일반 사용자는 이 섹션을 읽을 필요 없습니다. **저장소 관리자·IT 담당자**가 1회 설정합니다.

### 8-1. GitHub Secrets 등록

1. 저장소 → **Settings** → **Secrets and variables** → **Actions**  
2. **New repository secret**으로 아래 3개 등록:

| Secret 이름 | 설명 | 예시 |
|-------------|------|------|
| `OPENAI_API_KEY` | LLM API 키 (필수) | Gemini / Groq / OpenAI 키 |
| `OPENAI_BASE_URL` | OpenAI 호환 API URL | Groq: `https://api.groq.com/openai/v1` |
| `MODEL_NAME` | 사용 모델 | `gemini-2.0-flash`, `llama-3.3-70b-versatile` |

**Gemini 예시:**

```
OPENAI_BASE_URL = https://generativelanguage.googleapis.com/v1beta/openai/
MODEL_NAME = gemini-2.0-flash
```

**Groq 예시:**

```
OPENAI_BASE_URL = https://api.groq.com/openai/v1
MODEL_NAME = llama-3.3-70b-versatile
```

### 8-2. 워크플로 권한

- `daily-monitor.yml`, `monthly-report.yml`은 `contents: write` 권한으로 결과를 push합니다.  
- 저장소 **Settings → Actions → General**에서 workflow가 커밋할 수 있도록 허용되어 있어야 합니다.

### 8-3. 최초 동작 확인

1. Secrets 등록 후 **Daily Tech Market Monitor** → **Run workflow** (날짜 비움)  
2. 5~10분 후 `output/daily/`에 md 파일 생성 여부 확인  
3. 실패 시 Actions 로그에서 API 키·모델명 오류 확인

## 9. 로컬 PC와 함께 쓸 때

팀에서 **GitHub Actions + 로컬 Windows PC**를 병행하는 경우 흐름은 다음과 같습니다.

```text
08:00 KST  GitHub Actions — daily 생성·push
08:00 KST  로컬 PC (선택) — daily-catchup·push (로컬-only 보충)
08:20 KST  로컬 PC — git pull (GHA 결과 동기화)
```

| 역할 | 담당 |
|------|------|
| **GitHub만 사용** | §4~§5 — 브라우저에서 md 확인 |
| **로컬 PC 자동화** | [`사용설명서.md`](사용설명서.md) — Python 설치, `setup_scheduler.ps1` |
| **충돌 방지** | 양쪽 모두 push 시 `git pull --rebase` 후 push (워크플로·배치 파일에 포함됨) |

로컬 PC가 꺼져 있어도 **GitHub Actions만으로 데일리·월간 리포트는 계속 생성**됩니다.

## 10. 키워드·출처 바꾸기

설정 파일을 저장소에서 수정하면 **다음 Actions 실행**부터 반영됩니다.

### 10-1. GitHub 웹에서 편집

1. **`keywords.txt`** 또는 **`sources.txt`** 파일 열기  
2. 연필 아이콘 **Edit this file** 클릭  
3. 수정 후 **Commit changes** (main에 직접 또는 PR)

### 10-2. 키워드 (`keywords.txt`)

- **한 줄에 키워드 하나**  
- `#`으로 시작하는 줄은 주석  
- **맨 위쪽(주석 제외) 3개**가 AI 「키워드 관련도」 분석 기준

### 10-3. RSS 출처 (`sources.txt`)

- 형식: `이름 | RSS URL | 카테고리`  
- **현재:** 국내 ICT·정책·공공 R&D 중심 **34개** RSS + catch-up 시 korea.kr·PACST 아카이브  
- `sources.txt`가 없으면 `config/sources.yaml`의 **korean** 그룹만 사용

> **정부·R&D 타깃:** 국가계획·MOU·R&D 공고 등은 키워드 없이도 수집·요약 우선순위가 높습니다.

> 일반 사용자는 **키워드만** 바꿔도 충분한 경우가 많습니다.

## 11. 자주 묻는 질문 (FAQ)

### Q1. Python이나 Cursor가 필요한가요?

**아니요.** 리포트 읽기·다운로드는 **브라우저 + GitHub**만으로 가능합니다.

### Q2. 매일 몇 시에 올라오나요?

스케줄 기준 **08:00 KST** 전후입니다. GitHub Actions 지연으로 **08:15 전후**에 커밋이 보일 수 있습니다.

### Q3. 그날 기사가 없으면?

키워드에 맞는 **신규** 기사가 없으면 **md 파일을 만들지 않습니다.** Actions는 성공(✅)으로 끝나지만 커밋이 없을 수 있습니다.

### Q4. PC를 꺼 두어도 되나요?

**네.** GitHub Actions가 클라우드에서 실행합니다. 로컬 PC는 선택 사항입니다.

### Q5. 월간 보고서는 언제 올라오나요?

해당 월 **마지막 영업일** 08:30 KST 전후 (자동).  
중간에 필요하면 Actions에서 **Monthly Tech Market Report**를 수동 실행하세요.

### Q6. 데일리 md가 월말에 사라지나요?

- **GitHub Actions 월간 실행:** `--no-cleanup` 사용 → **데일리 md 유지**  
- **로컬 PC 월간 실행(기본):** 해당 월 md **삭제** (DB는 유지)  
  → GitHub에 이미 push된 md는 **커밋 히스토리**에서 계속 볼 수 있습니다.

### Q7. Word 월간 보고서는 없어졌나요?

기본 출력은 **`monthly_YYYY-MM.md` Markdown**입니다. GitHub Actions도 이 형식을 커밋합니다.

### Q8. 해외 뉴스가 안 보여요.

**정상입니다.** **국내 전용(Korea-only)** 모드 — 해외-only·외신 재인쇄 기사는 제외됩니다.

### Q9. 데일리 md에 영문 요약이 없어요.

**정상입니다.** 데일리·월간 모두 **한국어 R&D·투자 타겟팅** 중심입니다.

### Q10. 저장소 접근 권한이 없어요.

팀 관리자에게 GitHub 저장소 **Collaborator** 또는 **Organization** 멤버 초대를 요청하세요.

## 12. 문제 해결

### Actions가 ❌로 실패함

1. **Actions** 탭 → 실패한 run 클릭  
2. 빨간색 step 펼치기  
3. 흔한 원인:

| 오류 | 조치 |
|------|------|
| API key invalid | Settings → Secrets에서 `OPENAI_API_KEY` 확인 |
| Model not found | `MODEL_NAME`과 `OPENAI_BASE_URL` 조합 확인 |
| Push rejected | 동시 push 충돌 — 재실행하면 대부분 해결 |

### 08:00이 지났는데 새 daily가 없음

- 그날 **신규 기사 0건**일 수 있음 (정상)  
- Actions에서 **Daily Tech Market Monitor** 최근 run이 ✅인지 확인  
- **Run workflow**로 수동 재실행

### md는 있는데 월간 Markdown이 없음

- 아직 **마지막 영업일**이 아닐 수 있음  
- Actions → **Monthly Tech Market Report** → **Run workflow**로 수동 생성

### GitHub에서 표·링크가 깨져 보임

- GitHub Markdown 렌더러 한계일 수 있음  
- **Raw** 다운로드 후 Typora·VS Code 등으로 열기
