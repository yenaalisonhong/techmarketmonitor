## 1. 이 프로그램은 무엇인가요?

**Tech Market Intelligence Monitor**(이하 **모니터**)는 프라운호퍼 한국 사무소용 **기술 시장·R&D 타겟팅 모니터링 자동화 도구**입니다.

사람이 매일 뉴스·정책 사이트를 직접 돌아다니지 않아도, 프로그램이 다음 작업을 대신합니다.

| 단계 | 프로그램이 하는 일 |
|------|-------------------|
| 1. 수집 | 국내 RSS 피드(`sources.txt`)에서 기사·보도자료 목록을 가져옴 |
| 2. 보강 | 정부·공공기관 URL은 본문·첨부 PDF까지 추출해 요약 품질을 높임 |
| 3. 필터 | `keywords.txt` 키워드 매칭 + **정부·R&D 타깃** 기사는 키워드 없어도 통과 |
| 4. 요약 | AI(LLM)가 **투자 주체·목적·위탁 연구 니즈·접근 전략** 중심으로 한국어 요약 작성 |
| 5. 저장 | 데이터베이스에 저장하고, **데일리 리서치 로그**(`daily_YYYY-MM-DD.md`) 생성 |
| 6. 월말 | 한 달치 데일리 md를 모아 **국내 R&D 인텔리전스 월간 보고서**(`monthly_YYYY-MM.md`) 생성 |

**현재 모니터링 주제 예시:** 전력계통, 파워그리드, 스마트그리드, BESS, 데이터센터 전력, AI 인프라, 표준·R&D 정책, 투자·M&A 등  
→ 주제는 `keywords.txt` 파일을 수정하면 바꿀 수 있습니다.

## 2. 무엇이 만들어지나요?

설치·실행 후 아래 폴더에 결과물이 쌓입니다.

| 결과물 | 저장 위치 | 형식 | 용도 |
|--------|-----------|------|------|
| **데일리 R&D 인텔리전스 로그** | `output\daily\daily_YYYY-MM-DD.md` | Markdown(`.md`) | 매일 아침 팀이 읽는 **일일 브리핑** (R&D 기회 스캔 표 포함) |
| **월간 R&D 인텔리전스 보고서** | `reports\monthly_YYYY-MM.md` | Markdown(`.md`) | 월간 국내 R&D·협력 기회 요약 (적합 4점 이상) |
| **월간 Word TMR (선택·레거시)** | `reports\tech-market-report-YYYY-MM.docx` | Word | `.env`에 `GENERATE_LEGACY_TMR=1`일 때만 영문 생성 |
| **원본 데이터** | `data\monitor.db` | SQLite DB | URL 중복 제거·수리용 (일반 사용자는 직접 열 필요 없음) |
| **정부 계획 PDF 요약 (선택)** | `output\plans\` | 텍스트 | `summarize-plan` 수동 실행 시 |

### 데일리 로그 파일 이름 규칙

- `daily_2026-06-23.md` → **2026년 6월 23일** 하루치 수집·요약 결과
- Windows 탐색기에서 `output\daily` 폴더를 열면 날짜순으로 파일이 보입니다.

## 3. Cursor 없이도 사용할 수 있나요?

**네. Cursor는 전혀 필요하지 않습니다.**

Cursor는 개발자가 코드를 편집할 때 쓰는 도구일 뿐이며, 일상적인 활용은 아래만 있으면 됩니다.

| 필요한 것 | 용도 |
|-----------|------|
| **Windows PC** | 프로그램 실행 |
| **PowerShell** (또는 명령 프롬프트) | 최초 설치·가끔 수동 실행 |
| **메모장 / Word / 브라우저** | 결과물 읽기 |
| **LLM API 키** (Gemini, Groq, OpenAI 등) | AI 요약에 사용 |

**일반 사용자의 typical 흐름:**

1. **최초 1회:** PowerShell로 설치 + API 키 설정 + Windows 예약 작업 등록  
2. **매일:** 아무것도 하지 않아도 됨 → `output\daily` 폴더에서 md 파일 확인  
3. **월말:** `reports` 폴더에서 `monthly_YYYY-MM.md` 확인  
4. **가끔:** 키워드 수정, 수동 실행, 자료 보충

### 3-1. 로컬 작업 폴더 (OneDrive — Cursor 종료 후에도 동일)

**설정·키워드·결과물은 아래 폴더가 본진입니다.**

```
C:\Users\Admin\OneDrive - Fraunhofer\Documents\python-project
```

| 파일/폴더 | 여기서만 수정 |
|-----------|----------------|
| `keywords.txt` | ✅ (상위 3줄 = 매일 AI 분석 기준) |
| `sources.txt` | ✅ |
| `.env` | ✅ (API 키) |
| `output\` | ✅ (데일리·월간 결과) |
| `data\` | ✅ (DB) |

**코드(GitHub) vs 로컬 설정**

| 구분 | 방법 |
|------|------|
| 프로그램 코드 업데이트 | `run_sync_from_github.bat` 또는 `git pull` |
| Cursor/Documents → OneDrive 코드 복사 | `sync_code_to_onedrive.ps1` (keywords·output·.env는 OneDrive 유지) |
| 키워드·RSS·API 키 | OneDrive 폴더에서 메모장으로 직접 수정 |

**경로가 두 개일 때 (Documents vs OneDrive)**  
PC에 `Documents\python-project` 복사본이 따로 있으면 **다른 `keywords.txt`를 읽을 수 있습니다.** 한 번만 아래 중 하나를 실행하세요.

1. **권장 — junction 통합** (두 경로가 같은 폴더를 가리킴):
   ```powershell
   Set-ExecutionPolicy -Scope Process Bypass
   cd "C:\Users\Admin\OneDrive - Fraunhofer\Documents\python-project"
   .\setup_onedrive_junction.ps1
   ```
2. **확인 — 지금 어떤 파일을 쓰는지:**
   ```powershell
   cd "C:\Users\Admin\OneDrive - Fraunhofer\Documents\python-project"
   python -m src.main show-config
   ```
   출력의 `keywords_path`와 `analysis_keywords_top3`가 편집한 `keywords.txt`와 일치해야 합니다.

3. **Cursor 종료 후 junction 설정** (폴더 잠금 시):
   - `run_junction_after_close.bat` 더블클릭 → Cursor 완전 종료 후 실행

## 4. 처음 설치하기 (1회만)

### 4-1. 사전 준비

1. **Python 3.11** 설치  
   - [python.org](https://www.python.org/downloads/)에서 다운로드  
   - 설치 시 **"Add Python to PATH"** 체크 필수

2. **LLM API 키** 준비  
   - 예: Google Gemini, Groq, OpenAI 등 OpenAI 호환 API  
   - 팀에서 발급받은 키를 `.env` 파일에 넣습니다 (아래 4-3 참고).

3. 프로젝트 폴더 위치 확인  
   - 기본 경로: `C:\Users\Admin\OneDrive - Fraunhofer\Documents\python-project`  
   - 다른 PC에 복사했다면 해당 경로를 기억해 두세요.

### 4-2. PowerShell 열기

1. **시작 메뉴** → **PowerShell** 검색 → **Windows PowerShell** 실행  
2. 프로젝트 폴더로 이동:

```powershell
cd C:\Users\Admin\OneDrive - Fraunhofer\Documents\python-project
```

> **팁:** 탐색기에서 `python-project` 폴더를 연 뒤, 주소창에 `powershell`을 입력하고 Enter를 눌러도 같은 폴더에서 PowerShell이 열립니다.

### 4-3. 가상환경 만들기 및 패키지 설치

아래 명령을 **순서대로** 실행합니다.

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

- `Activate.ps1` 실행이 막히면 (보안 정책 오류):

```powershell
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass -Force
.\.venv\Scripts\Activate.ps1
```

- 성공하면 프롬프트 앞에 `(.venv)`가 표시됩니다.

### 4-4. API 키 설정 (.env 파일)

1. 탐색기에서 `python-project` 폴더의 **`.env.example`** 파일을 찾습니다.  
2. 복사해서 이름을 **`.env`**로 바꿉니다.  
3. 메모장으로 `.env`를 열고 API 키를 입력합니다.

**Gemini 예시:**

```env
OPENAI_API_KEY=여기에_실제_API_키_입력
OPENAI_BASE_URL=https://generativelanguage.googleapis.com/v1beta/openai/
MODEL_NAME=gemini-2.0-flash
```

**Groq 예시:**

```env
OPENAI_API_KEY=여기에_Groq_API_키
OPENAI_BASE_URL=https://api.groq.com/openai/v1
MODEL_NAME=llama-3.3-70b-versatile
```

> **주의:** `.env` 파일은 API 키가 들어 있으므로 이메일·채팅으로 공유하지 마세요.

### 4-5. 설치 확인 (테스트 실행)

가상환경이 활성화된 PowerShell에서:

```powershell
python -m src.main daily-catchup
```

- 처음 실행은 RSS 수집 + AI 요약으로 **수 분** 걸릴 수 있습니다.  
- 끝나면 `output\daily\` 아래에 `daily_YYYY-MM-DD.md` 파일이 생겼는지 확인합니다.

## 6. 수동으로 실행하기

스케줄러 없이, 필요할 때 PowerShell에서 직접 실행할 수 있습니다.

### 6-1. PowerShell 준비 (매번)

```powershell

```

### 6-2. 자주 쓰는 명령

| 목적 | 명령 | 설명 |
|------|------|------|
| **일반적인 데일리 실행 (권장)** | `python -m src.main daily-catchup` | 수리 후 마지막 완료일 다음날 ~ 오늘까지 **빠진 날짜를 모두** 생성 |
| md/DB 불일치만 수리 | `python -m src.main daily-repair` | LLM 없이 불일치 복구 |
| DB 날짜 정렬 + md 재생성 | `python -m src.main daily-refresh` | LLM 없이 전체 md 재빌드 |
| 어제 하루만 실행 | `python -m src.main daily` | 1회만 실행 (기본: 어제 날짜) |
| 특정 날짜 1회 실행 | `python -m src.main daily --date 2026-06-20` | 해당 날짜로 md·DB 저장 (수집 창은 실행 시점 기준 최근 24시간) |
| 정부 계획 PDF 요약 | `python -m src.main summarize-plan --file 경로.pdf` | PDF 요약 → `output/plans/` (데일리 24h 수집과 별도) |
| 이번 달 월간 보고서 | `python -m src.main monthly` | `monthly_YYYY-MM.md` 생성 (R&D 적합 4점+) |
| 특정 월 월간 보고서 | `python -m src.main monthly --year 2026 --month 6` | 2026년 6월 보고서 |
| 기간 재처리 (규칙 변경 후) | `python -m src.main daily-reprocess --from 2026-06-21 --to 2026-06-29` | LLM 재호출·DB·md 덮어씀 |
| 기간 완전 재수집 | `python -m src.main daily-reprocess --from … --to … --fresh` | 기존 DB·md 삭제 후 재수집 |
| 설정 경로 확인 | `python -m src.main show-config` | keywords 경로·상위 3 키워드 JSON |
| 데일리 md 유지하며 월간 생성 | `python -m src.main monthly --no-cleanup` | 월간 생성 후 daily md **삭제 안 함** |

### 6-3. `daily` vs `daily-catchup` — 어떤 걸 쓸까?

| 상황 | 추천 명령 |
|------|-----------|
| 평소 매일 실행 | `daily-catchup` |
| PC를 며칠 꺼 두었다가 다시 켬 | `daily-catchup` (빠진 날짜 자동 보충) |
| 딱 하루만 다시 돌리고 싶음 | `daily --date YYYY-MM-DD` |

## 7. 결과물 읽는 방법

### 7-1. 데일리 리서치 로그 (`daily_YYYY-MM-DD.md`)

파일을 열면 대략 아래 순서로 구성되어 있습니다.

#### (1) 상단 메타데이터

```text
날짜: 2026-06-23
기록자: Tech Market Monitor (auto)
총 항목 수: 22건 (기사 7 / 논문 15)
신뢰도 분포: A 0건 / B 22건 / C 0건
```

- **총 항목 수:** 그날 키워드에 걸려 AI 요약까지 된 기사·논문 수  
- **신뢰도:** A(최상) / B(보통·프리프린트 등) / C(참고용, 자동 생성 시 거의 없음)

#### (2) 오늘의 R&D 인텔리전스 요약

- 항목이 **1건 이상**일 때 생성됩니다.  
- **모니터링 키워드**, **오늘의 공통 흐름**, **R&D 기회 스캔** 표(적합도·고객 타겟·연계 포인트)가 들어 있습니다.  
- **R&D 적합도**는 1–5점 — **4점 이상**이 월간 보고서 포함 기준입니다.  
- **국내 R&D 타겟 시사점 (프라운호퍼)** 섹션: 국내 투자 주체 관점 요약.  
- **바쁠 때는 R&D 기회 스캔 표만** 읽어도 협력 타겟 파악에 충분한 경우가 많습니다.

#### (3) 항목 기록 (개별 기사·보도자료)

각 항목은 **`HH:MM 제목`** 형식(발행 시각 기준)으로 정렬됩니다.

## 12. 자주 묻는 질문 (FAQ)

### Q1. Cursor를 설치해야 하나요?

**아니요.** 설치·스케줄 등록·결과 확인 모두 Windows + PowerShell + 탐색기 + Word로 가능합니다.

### Q2. 매일 몇 시에 결과가 나오나요?

스케줄러 등록 시 **08:00**에 `daily-catchup`이 실행됩니다.  
RSS·AI 요약량에 따라 **08:00~08:15** 사이에 `output\daily`에 파일이 생깁니다.

### Q3. 그날 기사가 하나도 없으면?

키워드에 맞는 **신규** 기사가 없거나, 이미 DB에 저장된 URL만 있으면 **md 파일을 만들지 않습니다.**

### Q4. 하루에 최대 몇 건 요약하나요?

기본 **30건** (`MAX_ARTICLES_PER_RUN`). Groq 무료 한도 등을 고려한 설정입니다.  
`.env`에서 `MAX_ARTICLES_PER_RUN=50` 처럼 변경 가능합니다.

### Q5. API 비용이 드나요?

LLM(Gemini, Groq 등) **API 사용량에 따라** 과금 또는 무료 한도가 적용됩니다.  
팀에서 발급한 키·요금제를 확인하세요.

### Q6. GitHub Actions만 쓰고 로컬 PC는 꺼도 되나요?

저장소에 GitHub Actions 워크플로가 설정되어 있다면, 클라우드에서도 데일리·월간이 돌아갈 수 있습니다.  
다만 **로컬 `output\daily` 폴더**와 GitHub 저장소 동기화 여부는 팀 설정에 따릅니다.  
로컬만 쓸 경우 PC 또는 스케줄러가 돌아가는 서버가 필요합니다.

### Q7. 월간 보고서 후 데일리 md가 사라졌어요.

`monthly` 명령 **기본 동작**입니다 (로컬 PC). GitHub Actions 월간 워크플로는 `--no-cleanup`으로 md를 유지합니다.  
GitHub에 이미 push된 md는 커밋 히스토리에서 계속 볼 수 있습니다.  
md를 보존하려면 `--no-cleanup` 옵션을 사용하세요.

### Q8. 해외 뉴스가 안 보여요. 정상인가요?

**정상입니다.** 2026-07부터 **국내 전용(Korea-only)** 모드입니다. 해외-only·외신 재인쇄 기사는 수집·요약에서 제외됩니다.

### Q9. R&D 적합도 1–5점이란?

AI·휴리스틱이 매기는 **프라운호퍼 협력 적합도**입니다. 국내 투자 주체·위탁 R&D 니즈·제안 R&D 영역 등을 반영하며, **4점 이상**만 월간 보고서에 포함됩니다.

### Q10. Word 월간 보고서는 없어졌나요?

기본 월간 출력은 **`monthly_YYYY-MM.md` Markdown**으로 바뀌었습니다. 영문 Word TMR이 필요하면 `.env`에 `GENERATE_LEGACY_TMR=1`을 설정하세요.

### Q11. 정부·R&D 타깃이란?

국가표준기본계획, 과기정통부 보도자료, MOU·R&D 사업 공고 등 **프라운호퍼 협력 관점**에서 중요한 공식 발표입니다. 키워드에 걸리지 않아도 수집·요약 우선순위가 높습니다.

### Q12. 영문·한국어 요약이 다른데 정상인가요? (레거시 Word TMR)

**해당 없음(기본).** 현재 월간 보고서는 Markdown 단일 형식입니다. `GENERATE_LEGACY_TMR=1`로 영문 Word를 켠 경우에만 해당합니다.

## 13. 문제 해결

### "python을 찾을 수 없습니다"

- Python 재설치 시 **Add to PATH** 체크  
- 또는 `py -m venv .venv` / `py -m src.main daily-catchup` 사용

- `.env` 파일이 `python-project` **루트**에 있는지 확인  
- API 키 앞뒤 공백·따옴표 없이 입력  
- `OPENAI_BASE_URL`, `MODEL_NAME`이 사용 중인 서비스와 일치하는지 확인

### 스케줄러는 돌았는데 파일이 없음

1. 작업 스케줄러 → `TechMarketMonitor-Daily` → **기록** 탭에서 오류 코드 확인  
2. PowerShell에서 수동 실행해 에러 메시지 확인:

```powershell
cd C:\Users\Admin\OneDrive - Fraunhofer\Documents\python-project
.\.venv\Scripts\python.exe -m src.main daily-catchup
```

3. `.venv` 경로가 `setup_scheduler.ps1`의 `$PROJECT`와 일치하는지 확인 (PC마다 경로 다르면 스크립트 수정 필요)

### 요약 품질이 이상함 / 특정 날 다시 만들고 싶음

관리자·개발 담당자에게 **`regen_daily_report.py`** 실행을 요청할 수 있습니다.

```powershell
python regen_daily_report.py 2026-06-21
```

→ DB에 있는 해당 날짜 기사를 AI로 **재요약** 후 md 재생성.

### Groq 등 무료 한도 초과

- `.env`에서 `MAX_ARTICLES_PER_RUN`을 줄이거나  
- `SUMMARIZER_REQUEST_DELAY=2.0` 으로 요청 간격 늘리기  
- 또는 다른 API 키·모델로 전환
